## Model ID: WARDLEY_MAPPING

## Definition and Core Principle
*   Wardley Mapping is a strategic planning technique that maps user needs, the value chain fulfilling those needs, and the evolutionary stage of each component within that chain.
*   Its core principle is that everything evolves from uncharted (Genesis) to custom-built, to product/rental, to commodity/utility, enabling strategic decision-making based on this understanding.

## Concrete Examples and Analogies
*   **Cloud Computing:** Before AWS, companies built their own data centers (custom-built). AWS commoditized compute and storage, allowing businesses to focus on higher-value activities. A Wardley Map would show "compute" moving from custom-built to commodity.
*   **Car Manufacturing:** A car company might map components like engines (commodity), infotainment systems (product), and a new AI-driven autonomous driving feature (genesis/custom-built), guiding decisions on sourcing, development, and investment.

## Common Pitfalls and Anti-Patterns
*   Treating the map as a static artifact rather than a dynamic, evolving landscape.
*   Falling prey to the "not invented here" syndrome by custom-building components that are already commoditized, or conversely, attempting to commoditize something that is still in its genesis stage.
*   Focusing solely on internal capabilities without understanding the broader market evolution and competitive landscape.

## Logic Gate
*   The problem involves strategic decision-making regarding product development, service delivery, or organizational structure.
*   There is a need to visualize and understand the dependencies within a value chain, from user needs down to underlying components.
*   The situation requires assessing the evolutionary stage (e.g., genesis, custom, product, commodity) of various components to identify strategic opportunities or threats.
*   The goal is to determine where to innovate, where to standardize, where to outsource, or where to invest resources for competitive advantage.

## Thinking Steps
1. **Identify the Anchor:** Define the user and their core need.
2. **Map the Value Chain:** List the components required to fulfill that need, linking them by dependency.
3. **Determine Evolution:** Place each component on an evolutionary axis: Genesis, Custom Built, Product (+Rental), Commodity (+Utility).
4. **Identify Strategic Moves:** Look for components that are evolving, areas to out-innovate competitors, or areas to commoditize to save costs.

## State Output
Generate a visual map of the value chain against evolutionary stages and highlight strategic opportunities.

## Termination Criteria
Provide a clear strategic action plan based on the map (e.g., 'Commoditize X, custom-build Y').
