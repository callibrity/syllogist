## Model ID: PLATFORM_VS_PRODUCT

## Definition and Core Principle
*   This model distinguishes between two fundamental business strategies: building a direct solution for end-users (Product) versus creating an ecosystem that enables others to build solutions (Platform).
*   A Product focuses on solving a specific problem for a specific user segment, while a Platform provides infrastructure, tools, and rules for multiple parties to interact and create value, often leveraging network effects.

## Concrete Examples and Analogies
*   **Product:** A SaaS accounting software (e.g., QuickBooks) directly serves businesses with financial management tools. A mobile game (e.g., Candy Crush) provides direct entertainment to players.
*   **Platform:** Apple's App Store provides infrastructure for developers to build apps and for users to discover them. Airbnb connects hosts with guests, providing tools and a marketplace for both sides.

## Common Pitfalls and Anti-Patterns
*   **Pitfall:** Trying to be both a product and a platform simultaneously without clear strategic separation, leading to diluted focus, resource allocation conflicts, and confusion for users and partners.
*   **Anti-pattern:** Building platform-like features (e.g., APIs) without a clear strategy for attracting and supporting third-party developers, or conversely, building a product that inherently needs network effects but treating it as a standalone, linear product.
*   **Bias Counteracted:** The tendency to default to a "product-first" mindset even when a platform strategy might be more scalable, defensible, or appropriate for the problem space.

## Logic Gate
Apply this model when the problem description involves:
*   Analyzing or defining the fundamental business model or technical architecture of an entity.
*   Making a strategic decision between directly serving end-users with a complete solution (Product) versus enabling third parties to create value and interact within an ecosystem (Platform).
*   Addressing challenges related to scalability, network effects, or ecosystem development within an existing business.
*   Evaluating whether a current offering is struggling due to a misaligned product vs. platform strategy.

## Thinking Steps
1.  **Analyze the Value Creation:** Does the company capture value by directly serving user needs (Product) or by facilitating interactions between producers and consumers (Platform)?
2.  **Assess Network Effects:** Does the value of the offering increase as more people use it? (Critical for platforms).
3.  **Evaluate Extensibility:** Is the core technology designed to be built upon by third parties via APIs or SDKs?
4.  **Choose the Focus:** Explicitly decide whether resources should go toward building features (Product) or building the ecosystem/infrastructure (Platform).

## State Output
Articulate the chosen strategy, detailing how value is created and whether the focus is on end-users or an ecosystem.

## Termination Criteria
Align organizational structure, technical architecture, and go-to-market strategy with the chosen platform or product approach.
